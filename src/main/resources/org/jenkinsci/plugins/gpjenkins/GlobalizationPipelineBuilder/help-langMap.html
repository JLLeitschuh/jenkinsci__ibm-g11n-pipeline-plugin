For Upload and Download goal

A JSON representation of key:value pairs
where each Key is is a BCP 47 language tag used by Globalization Pipeline service instance. And the element Value is a language ID used for output resource file or path name.

  • Upload: Uploads goal is only concerned about the Key of the json. Those are the target languages you want to choose for your source files to be translated in.

    • For example

    { "de":"I am", "es":"Not affected", "fr":"by upload goal" }

    will upload your source files to IBM Globalization Pipeline for translation in German, Spanish and French.

  • Download: Downloads goal is concerned about both Key and Value of the json. Keys are the target languages you want to choose for downloading your translated files from Globalization Pipeline to Jenkin's output Directory. Values are the language ID used for output resource file or path name

    • For example

    { "de":"de", "es":"Spanish", "fr":"Parth" }

    will download translated files from IBM Globalization Pipeline for translation in German, Spanish and French with file name
    <filename><BCP47 formatting>de.<type>
    <filename><BCP47 formatting>Spanish.<type>
    <filename><BCP47 formatting>Parth.<type>
    same naming rules are mapped for output directory creation if you chose LANGUAGE_DIR or LANGUAGE_SUBDIR in Download goal's output layout

Note that in download goal, if Key (target language you opt for downloading in Jenkins workspace) is not set in Globalization Pipeline through upload goal or manually by UI dashbaord, those target languages won't be downloaded.
